    HookMethod is unnecessary in plugins. Any method in a plugin is a "hook" and can be called (using Plugin.Call) by other plugins without manually declaring them as a hook.

    There are examples in existing threads on the forums on how to call a hook in a plugin as well as in the Docs. Generally you'd set a [PluginReference] attribute for that plugin and use .Call on the plugin instance that is set using by that reference or from plugins.Find.
