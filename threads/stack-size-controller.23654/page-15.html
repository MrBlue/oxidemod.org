    Downloading an earlier version would be bad, since then every server owner would also have to downgrade and miss out on other fixes and features.

    I have sent a bug report to facepunch that includes the necessary changes in client and server. We'll see if they decide to do something.

    This is what i sent:

    When splitting item stacks, if the split amount is above 65535, the actual split amount will be equal to the overflow value of casting int to ushort. (200 000 would be 3392, this matches current behaviour in game)

    Would it be possible to implement a fix for this? The fix is rather easy and shouldn't take more than a few minutes at most.

    I looked at the decompiled assembly for both the client and server, and the only changes would be the following:

    CLIENT:
    LocalPlayer.cs
    Method: MoveItem(uint itemid, uint targetContainer, int targetSlot, int amount)
    Fix: Change ServerRPC call, remove casting amount to ushort. Also may or may not want to check if amount is > 0 since we're not casting to unsigned anymore (this seems to be checked on the serverside anyway, so it doesn't matter much). If not, don't send RPC call.

    SERVER:
    PlayerInventory.cs
    Method: MoveItem(BaseEntity.RPCMessage msg)
    Fix: Change reading amount as UInt16 to Int32() and remove casting to int.
