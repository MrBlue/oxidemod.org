  12. NobodyFTW

    SkinN õ.Õ'.|. updated Notifier with a new update entry:

    Notifier | Improvements, Tweaks and Fixes!

    [ CHANGELOG ]
    - Added arguments to rules commands, so you can get a speciefic language. (EXP /rules fr would get the French language)
    - Admin tags will be added at plugin load and on player connect, and will be removed on plugin unload and player disconnect, properly. (Still needs some tests though)
    - Prefix is also formated along with the message to be sent.
    - Changed the way to format colors.
    - Improved color format system.
    - Fixed IP and Port misplacement formats on Adverts.
    - Fixed Chat...
    Click to expand...
    [ NOTES ]
    * Color formats are now a bit similar to HTML, but simpler:
    EXP 1:
    Code:
    {% raw %}
    <red>This message is red<end>
    {% endraw %}
    OUTPUT: This message is red
    EXP 2:
    Code:
    {% raw %}
    <red>This message is red and <#0000FF>blue<end><end>
    {% endraw %}
    OUTPUT: This message is red and blue
    * You can now format prefix color as you would format a message:
    EXP:
    Code:
    {% raw %}
    "PREFIX": "<red>O<end><#00FF00>MEGA<end>",
    {% endraw %}
    OUTPUT: OMEGA


    [DOUBLEPOST=1427599226][/DOUBLEPOST]I am sorry I had to change the color formats once again, but this time is definitive I promise, reason I changed it is because there was still was a few things you could not write in messages for exp:
    Code:
    {% raw %}
    "This is our IP: :lime:{ip}:{port}:/end:"
    {% endraw %}
    This would mess up because Notifier would change {port} as it was a color since it's between two dot characters.
    This new format it's way more neat and won't bring much trouble, as far as I test out, it's similar to HTML but simpler.

    Again I'm sorry for the trouble I caused you guys having to change all the colors in the config file, but it's for the greater good.
