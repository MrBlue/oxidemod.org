    SkinN õ.Õ'.|. updated Notifier with a new update entry:

    Notifier | Disconnected messages spam fix attempt

    [ CHANGELOG ]
    - Attempt to fix disconnected messages spamming the chat while users are trying to connect to the server and are rejected in the connection process.
    - Moved Connected message to OnPlayerInit to make sure the player is fully connected to the server before sending the message.
    Click to expand...


    I can't promise this will solve the issue, but it should be a way around it. I'm still working on a new feature so you are able to format {reason} into the disconnected messages, to display for what reason the player left the server, and when this is done it will be more easy to find whether or not the player has been rejected on the server connection process.
    But I'm pretty sure this fix reduces the spam a lot, and significantly on high populated servers since it has more traffic when it comes to players connecting and disconnecting.
