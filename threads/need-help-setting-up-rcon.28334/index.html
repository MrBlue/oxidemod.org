  6. nanohard

    There are "internal IPs" and "external IPs".
    An internal IP usually takes the form of 192.168.x.x or 10.0.x.x but can be different; those are just the two most common sets. For example, if you have a router (like most end-users) then all of your devices that are connected to that router will have an internal IP. This network is called a "subnet". Each section of numbers is 8 bits and there are 4 sections so there are 32 bits in total; if all of my devices were on the subnet of 192.168.0.0/24 then I can have as an IP 192.168.0.x where "x" is any number between 1 and 255 (because 8 bits can go up to integer 255). The number after the slash is how many of the bits do not change. So if the number is 24 then only the last 8 bits can change; if the number is 32 then none of the bits can change.

    The router takes all of those IP addresses and "routes" them through one "external" IP. The external IP is determined by your Internet Service Provider (ISP) and is usually dynamic, which means that it can change every so often without your knowledge. When you typed into Google "what is my IP", that is your external IP address. When two computers are talking to each other (there is always a client and a server), let's say a website (remote server) and your desktop (local client) which is behind a router, that server will use your external IP to send you data and your router says, "Oh, the local client that initiated this conversation is this desktop here, not the phone in the other room. I know this because the client that initiated this conversation has the IP address 192.168.0.34 which is the desktop computer. So I'll send the data from that conversation to the desktop, not the phone."

    So, in what you did above, typing "0.0.0.0" is actually a synonym for 0.0.0.0/0, which means that it can be anything (x.x.x.x); if you had more than one IP address on that computer it would complain and say "Hey man, pick one" which is when you would enter "192.168.0.34" or whatever your internal IP is for the Network Interface Card (NIC) on that computer that you wanted to use (which either the router assigns for you [dynamic/DHCP] or you assign yourself [static]).

    You want to use the internal IP for both the game and RCON. The port can be anything that isn't already used; valid ports are from 1-65535 (a 16-bit integer). Usually the lower end of the port range (0-1023) shouldn't be messed with or used, as they are commonly used for system operations (SSH, HTTP, FTP, Samba, etc). The port for RCON does not need to be "+2" of the game, it just needs to be different because 2 services on the same computer can't use the same port, but they can use the same IP address. Pro tip: it's always best to stick with defaults and standards unless you can't, so 28015 for the game server and 28016 for RCON, but you don't have to; it just makes life easier.

    If you wanted to host a server from a computer behind a router, then computers on the outside (remote clients) would use your external IP address and a port to connect to you, but the router wouldn't know where to send that data! That's why "port forwarding" is a thing. When data hits your router it needs to know where to go. So if you're hosting a server behind a router you need to forward the port 28015 on the router to port 28015 on your computer. That way if a remote client says, "I want to talk to 297.84.13.235 on port 28015" then the internet (DNS server, but that's not important here) says, "Oh, that IP belongs to this house, I'll forward the data to them for you." But if you didn't forward port 28015 from the router to one of your computers, the router will be like, "I'm lost and there are no road signs. Guess I'll just throw this super-important message in the trash can right here..."

    Hopefully that answers your question.
