    That's a terrible implementation. You're only checking the first element in that list, which is not at all scalable. Secondly, your return value is non-null if the player isn't an admin and the connection isn't null, always, which will prohibit the user from executing the command regardless of it's value. A proper solution would return a non-null value if the list contains the command.
    Code:
    {% raw %}
    return /* this name needs some work */ blackCmd.Contains(arg.cmd?.Name) ? false : (object)null;
    {% endraw %}
