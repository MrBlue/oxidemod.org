  15. Mughisi

    Loup-des-Neiges said: ↑
    Players harassing me because it often happens that the 'TPR' teleport a bad player.
    Click to expand...
    It is a fact that TPR had some major flaws before version 1.0.0 which allowed the player that send out the request to also accept the request, however this has been fixed in 1.0.0.

    The teleport request is saved temporarily in the plugin's memory, this saved data contains the player object of the player that has send the request and the steam id (unique!) of the player that he requested the teleport to. When a player uses the tpa command it will look for a match against the steam id, if it doesn't find a match then there is no request pending for that player, if there is a match then a player has requested a teleport and it will initiate the teleport for that player that was saved earlier. Because of this it should actually be impossible for others to accept the teleport now and it should be impossible that the wrong player is teleported.

    I've asked a few server admins that I know that are using the plugin and they say it's working perfectly for them. Now the question is, is it really teleporting the wrong player, or is it teleporting to the wrong location or not teleporting at all? And are you using the latest update?
