    oxide.directory being set to "" is actually currently unhandled and creating the entire oxide directory structure inside the server's directory structure without the actual oxide folder which is unintended. Because of this the oxide.config.json file is generated in the old location which makes the core think it is an old config and removing it every time you launch the server. For now an easy solution would be to just not specify -oxide.directory and let the oxide folder be generated in the server's root directory.
