  6. elninost0rm

    You have to make a moderator group within Oxide and then grant specific permissions to that group rather than using the "ownerid" console commands within Rust. As stated above, a moderator is treated the same as an admin for no-clip / fly / etc. purposes, so you basically have to add them to a group within Oxide and then assign them specific permissions for certain plugins.

    This is good for kick / ban / mute, given that all of those functions have a plugin, but it falls short for giving them the ability to TP, since that plugin does not use permissions (yet).
