    Code:

    void OnItemAddedToContainer(ItemContainer container, Item item)
{
   Puts(item.GetOwnerPlayer().displayName + " has added " + item.amount.ToString() + "x " + item.info.displayName.english + " to the container with the id " + container.uid.ToString())
}

    Well, i fiddled a little more with this.

    At place where the hook is currently placed is no possibility to get information about previous container, previous owner has removed beforehand.

    Item.MoveToContainer is too complex to put hook in right place and to have all required and correct information.
    For instance, OnItemAddedToContainer won't be called if you merge two stacks, take this into account!

    Uhm, well, it's still a "kludge" but it is a shorthand way to get what we need and does not require any additional dictionary for tracking.
