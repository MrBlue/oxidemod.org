    I'm with SS too, they don't auto-steam update, they let you control that yourself (which is good). The order to do things:

    - Backup map/player data
    - Steam Update
    - Uninstall Oxide
    - Install latest Oxide
    - Uninstall/Install plugins which rely on latest oxide
    - test

    Unfortunately Rust updates make this sort of thing necessary - they often change code that then 'breaks' important plugins like oxide, which in turn affects other mods that rely on oxide. It's all in the name of improving the game though!

    Best to hold off steam updates each time until oxide and any plugins your server relies on (e.g., Loot Lists and Crafting Controller if your server has 'rare' settings, that sort of thing) - it means your server won't show up in the list for players who have updated their client already, but that's usually better than munging up the balance of your server.[DOUBLEPOST=1390611943][/DOUBLEPOST]
    Tantrumus said: ↑
    My server world data was wiped 3 seperate times since last night only updating this single plugin, each time. No other changes. Fortunately I save/backup a lot so the players didn't revolt on me.
    Click to expand...
    This will happen if you 'Stop' the server after a 'Start' or 'Restart', before the world data has been loaded.

    Easiest way to avoid wiping your map like this (which has nothing to do with this plugin, btw) is to connect at least once to the server before ever hitting the Stop button. Note that 'Steam Update', 'Restart', backup functions, and possibly a couple of other server actions, all initiate a 'Stop' as well, so can be destructive in this manner. If for whatever reason you can't login each time before a Stop, at least open up your server's 'Web Console' if you have one available to you, and wait for the 'rust_island_2013.sav loaded' line to come up (I think it's soon after 'finished unpacking trees!').

    The more you know..
