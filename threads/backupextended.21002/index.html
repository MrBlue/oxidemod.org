    Fujikura submitted a new resource:

    Backup Extended - Backup extension with customizable number of retention folders

    This plugin extends the inbuilt backup processing.

    It does use the same folders like the inbuilt function, but can be used with x numbers of folders.

    The plugin can be configured to run an automated server backup on server startup.
    It has also in inbuilt timer function.

    Backups can be configured to broadcast the backup process into the server chat, with an optional delay between announcents and the processing of backups.

    nope^^
    It extends the normal backup functionality ( server.backup ).
    This function has a limitation to 4 savepoints only, which made it hard to keep frequent backups without a to high time-space between.
    And there's also no inbuilt timer function and no feedback like "backup complete".'

    This plugin fullfils ALL those needs, as you can set up a retention( revision) by any number you might need (number of backps), and you can set it to auto-backup on each server-start...and so on
