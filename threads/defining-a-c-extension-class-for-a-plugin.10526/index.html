  5. Wulf Community Admin

    @bawNg mentioned this when it was previously asked:
    CSharp plugins are currently designed to be implemented in the Oxide.Plugins namespace. The API requires access to this namespace, and since plugins are limited to single files, there is very little advantage to using a separate namespace.

    I know that there is a need for decreased overhead of inter-plugin communication as well as clean implementation of compartmentalized plugins. Options are currently rather limited but there will be more available in the future. Until then plugins will just have to live with using the Oxide.Plugins namespace.

    Various things are going to change in the future, as more options for large scale plugins are made available to developers. There is a lot of room for improvement in this department and there are multiple possible directions which can be taken, some of which are already currently in the early stages of being prototyped and benchmarked.
