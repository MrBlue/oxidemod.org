    Dear community, coders,

    Below you'll find my private plugin request. I tried my best to make it as informative as possible. If anything is missing, please let me know!
    If anyone is interested in working on this with me (or rather for me) I would like your price and maybe extra info (or even extra ideas to extend my request) in private message or posted in this topic.

    Random Treasure

    Explanation:
    This plugin should be mainly a way to have an interesting event happening on the server to try and draw players towards each other for some action.
    It would first announce that the event will start, in chat it says a random locked treasure box has been detected and it shares the coordinates in chat.
    Everyone is encouraged to engage on a trip because in <timeout> minutes the treasure will unlock and can be looted. It would be nice to have some visual effect like some fire or smoke coming from the location of the lootbox so it is easy to find.
    After the before mentioned set amount of minutes the crate 'unlocks' and players can open to retrieve the contents, once all loot is gone the chest disappears (like an airdrop kinda). The first person that opens the lootbox is mentioned in chat as the winner.
    If the chest does not get looted in time (within the timeout configured in the settings) it gets removed and the event gets cancelled without winner, this is also mentioned in chat.

    Example Configuration:
    • A list of possible spawn points (of the loot crate)
    • Minimum of players needed before trigger
    • Minimum amount of minutes since last time to trigger again
    • Maximum amount of minutes since last time to trigger again
    • Chat format, prefix and text (maybe in lang / translation)
    • Type of loot in spawned chest(s) and amount of filled slots in loot box.
    • Amount of seconds between announcement of event and unlocking of crate
    • Workshop skinid of the large wood box that functions as loot crate. (this way we can make it look like actual treasure ;])
    • Amount of seconds after the treasure was unlocked it should be removed (when not looted).

    Example commands:
    /th: Gives help and information about the plugin and it's commands.
    /th getloc: Print your current location in chat and in server console to make it easy to configure new spawnpoints for the loot crate. (Is overall a useful command for other purposes as well)
    /th start: Starts a random treasure hunt event on one of the locations configured in the settings.
    /th start here: Starts a random treasure hunt event on your current location.
    /th cancel: Cancels any currently running treasure hunt event. All effects and boxes spawned in this event gets removed.
    /th info: Show info about the current event (like: Time before unlock or time before event end. Time until next event if no event is currently running. Location of the current box if event is running).

    Example text:
    Prefix: [TreasureHunt] or [TH]
    EventStart: The Rust gods are pleased and have sent us a Treasure Box. It can be found at <location x y z> and will unlock in <timeout> minutes to unveil it's epic treasure!
    EventWin: Player <name> was able to loot the treasure box!
    EventRunout: Nobody was able to loot the treasure box in time, the Rust gods have reclaimed it in dissapointment.
    EventInfo: Treasure Hunt \n ------------------- \n Time until next: <timeuntilnext> \n Time until unlock: <timeuntilunlock> etc...


    Thanks for reading,

    iShot
