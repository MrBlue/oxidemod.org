  4. Wulf Community Admin

    Oxide doesn't control or manage if your server is listed or not, that is managed by the Steam, the Rust client, and Facepunch. The only thing Oxide does that is related to the server listing is that it adds the "modded" (if set to modded in the config) and "oxide" tags for Steam and the client to use, but that is standard with how Rust's client works. The Rust client uses that "modded" tag to list it under the Modded section. So installing Oxide 1000 times wouldn't really matter, it is all dependent on if Steam sees your server and the Rust clients get your server listing from Steam.

    As far as automatically updating Oxide goes, I would not recommend it.

    Hosts like to blame things, especially when they don't fully understand them.

    Rust's server listing is a bit finicky, but the listing is handled entire by Steam and your server, not Oxide. The only influence Oxide has over it is the tags (as described in the previous post) that are provided to Steam that are then used by the Rust client to categorize your server.

    Generally it is a matter of Steam recognizing that your server is there and then the Rust client getting that updated list of servers from Steam. Those tags (which include map info, player counts, etc.) are sent on server startup and periodically while the server is online and accessible by Steam. The Rust client may not see the changes immediately though, which is why it takes some time. By re-installing your server numerous times though, you're restarting and re-sending that information over and over to Steam. So yes, most likely the restart, but certainly not Oxide as like I said... it doesn't control or manage the communication with Steam at all.
    • Rust without Oxide ("vanilla") = no "modded" and "oxide" tags added, so Rust client lists the server under Community once Steam provides that
    • Rust with Oxide = "modded" and "oxide" tags added, so Rust client will list the server under Modded once Steam provides that
