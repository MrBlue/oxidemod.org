    If you delete the oxide.groups.data and oxide.users.data files while the server is offline, they would start fresh, but you'd lose all permissions assigned. You can make the 'player' group again manually, but that won't fix the players not in that group anymore.

    It's likely easier to start fresh and let all players get auto-assigned when they reconnect with the fresh perm files.
