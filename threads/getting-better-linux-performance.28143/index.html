  2. Silentwidow

    Not to be a snob here, but I've never heard of an E312, can you elaborate as to what CPU you actually have?

    Rust doesn't give much of a shit about core count, it's about core speed. Mine only uses 1-2 of my 12 cores, but it maxes those out. I average around 220-250FPS with 4 players on. And that's with a good bit of mods, including the navmesh.

    If you have a Xeon on a server motherboard, be sure you have the power saving options off, it will limit the CPU Frequency and drop your performance a lot.

    I am running a Dell T410 with a single X5660 running at 100% power, so it's 3.02GHZ at all times. I have 12GB of ram as well, Rust does like RAM though, uses about 5 GB of RAM.

    I also run Windows.

  3. Kappasaurus

    You can attempt running this command, it may help, though ultimately its success depends on your host.
    Code:
    {% raw %}
    for x in `ls /sys/devices/system/cpu/ | grep -E "cpu[0-9]+"`; do echo "performance" > /sys/devices/system/cpu/$x/cpufreq/scaling_governor; done
    {% endraw %}
