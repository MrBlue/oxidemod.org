            rust.SendChatMessage(player, "bumFights", player.transform.position.x, "0");
            rust.SendChatMessage(player, "bumFights", player.transform.position.y, "0");

                            break;

                            this.GameTracker--;
                            break;
                        }

            else

                    rust.SendChatMessage(playerj, "bumFights", "The game is full", 0)

    Your error is coming from player.Reply most likely, as that is not from Rust, it's from Oxide's Covalence API. You can't use normal command registration for that. So you'd likely want to use player.SendChatMessage or rust.SendChatMessage instead.

    for some reason the send chat message wouldn't work at first, error wanted 2 arguments.... my example had 4... so 4 is what you get
    Click to expand...
    You can have 2 or 4 arguments, either will work. The first needs to be a valid BasePlayer (player) and the second the message. The rest of the arguments are optional and are only used if you want to set a prefix (prefix would be second, message 3rd if using all arguments) or custom chat icon.
