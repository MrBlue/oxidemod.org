    I am using a VPS (just because I want to test some things). I am using my only internal IP address that I have available on eth0 and am trying to connect using the client to the external IP address; I have a website running on this VPS without issue.

    I cannot connect through Rust client or telnet in to my server on ports 28015 or 28016; iptables is set to ACCEPT on all policies.
    I am using LGSM and am defining my internal IP address in the correct location. I also tried without using LGSM the first go-around with the same result, so I'm thinking there's some sort of misconfiguration of the server that I'm missing. I have just under 4GB RAM, but the Rust server still starts, so I don't think that's why I can't connect.
    There is no client information in the Rust logs, so my issue has to be at the VPS connection level; not being able to telnet in makes me think that as well.

    Config details: rustserver - Pastebin.com
    Server log: rustserver - Pastebin.com
    netstat -nap | grep Rust : rustserver - Pastebin.com

    Thoughts?

    Edit:
    I've also stopped my web server just in case these two things weren't playing nicely- issue remains.

    Edit 2:
    nmap scan of ports shows 28015 and 28016 are filtered on TCP and open/filtered on UDP. That sounds correct.

    SOLVED:
    Stupid VPS putting their stupid firewall rules on my server, only accessible on their stupid GUI.
    Set ingress firewall rule to allow protocol and port, ie: tcp:28015-28016, udp:28015-28016
