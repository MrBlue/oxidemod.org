    The way this works is it reads http://api.steampowered.com/ISteamN...ppid=252490&count=1&maxlength=300&format=json and then parses for devblog and community update #'s and then stores the data in data directory. count = 1 which means it just grabs the last post. means that it'll only grab the last posted. if you see both 0 then the last news item posted contains no "community update" or "devblog".. it contains something else. play with the count # in that link and you'll see. I assume that over time the right numbers will be stored. in order to get both numbers initially populated, maybe increase the count in the cs file and then revert it afterwards. or edit your data file manually.

    Maybe plugin could load more data and then use the highest numbers it finds as a more permanent solution
