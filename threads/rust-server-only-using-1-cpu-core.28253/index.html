  3. Wulf Community Admin

    The Rust server generally does not use more than 1 or 2 cores because that is all that Unity supports. Unity is the engine that Rust uses. Any other cores that get used are generally from other tasks that Rust runs in the background or whatever to handle generating things and such (ie. navmesh).

    LGSM is not in itself its own process. It simply just compiles all of the scripts needed to install, run and update your Rust server.

    @AmazingAlex224 When a thread uses too much CPU it suspends the process and copies all of the registers into memory to load onto a new thread. This is normal and shouldn't cause any problems and if it does this is an issue with your servers load balancing function.
