  5. Kunazai

    This was driving me nuts that when pvpgod was set to true, it extended to sleepers as well, even though there is a separate sleepergod option. I figured out that if you change line 884:
    Code:
    {% raw %}
    else if (target != null)
    {% endraw %}
    to
    Code:
    {% raw %}
    else if (target != null && player.IsConnected())
    {% endraw %}
    In the code, it will check if sleeper god is enabled, and when you have it off, it just checks for pvpgod instead, and applies it to all player entities, sleeping or not. with my change, when it moves on to check for pvpgod, it ignores players who are not connected to the server. I did player.IsConnected() instead of player.IsSleeping() because when you use the n-teleport mod, you are sleeping when you use /town. player.IsConnected() circumnavigates the issue of being spawn camped.
     
    Kunazai, Feb 9, 2017
