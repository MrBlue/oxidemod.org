    Sounds like you're seeing a similar bug to what I had mentioned (zones affecting buildings outside their effective area).

    I narrowed it down to how ZoneManager handles MeshColliderBatches (a group of colliders attached to building objects) when a zone collider is triggered - when you place multiple buildings in close proximity they can get added to the same MeshColliderBatch, but when ZoneManager detects the MeshColliderBatch it simply adds all child entities in the batch to the zone, regardless of whether they actually intersect with the zone.

    @Nogrod Can you add a condition to OnTriggerEnter and OnTriggerExit in the collider loops to check for intersection before calling CheckCollisionEnter or CheckCollisionLeave? Something like:
    Code:
    {% raw %}
    if(instance.collider.bounds.Intersects(gameObject.GetComponent<Collider>().bounds))
    {% endraw %}

  13. ignignokt84

    @GST Kapper @Steenamaroo @Nogrod
    This is another known bug that I mentioned a while ago that's not been fixed "officially" yet, where box zones affect players in an area 2x the zone dimensions. This can be corrected by going into ZoneManager.cs, searching for "OverlapBoxNonAlloc" (around line 297), and changing the Info.Size parameter to Info.Size/2f:
    Code:
    {% raw %}
    entities = Physics.OverlapBoxNonAlloc(Info.Location, Info.Size/2f, colBuffer, Quaternion.Euler(Info.Rotation), playersMask);
                                                    // ^ here ^
    {% endraw %}
