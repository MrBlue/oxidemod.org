  15. GRoDT

    Here's a idea to fix some issues with servers that utilize the /remove tools. Currently there are 2 larger problems with game play with remove tool which I'll lay out and then I'll offer a solution. Some of which is already implemented some would need to be addressed and made possible if not already. I'll only list the known large problems not the positives of it in this case of course each version has it's own positive points.

    1. Only foundation owner can remove
    • Issue is when individuals builds outside a persons base that artifact if it failed will be there almost indefinitely unless manually destroyed/decay occurs.
    • Friends can't help /remove or live in base pending design without a share feature. (I'm believe that Reneb build this share feature into his older R-Remover tool maybe this can be reimplemented)
    2. Need access to all tool cupboards in range to remove
    • Individuals/groups will seek out bases and their cupboards and remove the WHOLE base which in some senses is the ultimate griefing
    • Unless the percentage is controlled the above problem is even more exacerbated by the fact people will do this on lower gather servers to get resources off /removing whole bases
    SOLUTION:
    1. Can take over a foundation if you leave a tool cupboard, assuming no other cupboards in there, on the foundation or building permission range for (x) hours. So if there is a random abandoned base outside you main base, you can eventually then own it to become yours to build or remove.

    The implementation of the share with friends feature would be nice as well.

    Other variations of this can make this tool work even better and i'm open to what your thoughts are and if this is possible.
