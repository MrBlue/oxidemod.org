    Come on, guys, try harder. It's in the config file, \oxide\config\RemoverTool.json
    Look at the bottom, around line 815 on my config file. There are five true/false options, all start with "Remove - Normal -Use" Pick the ones you want. If you want anyone authorized on a TC to be able to remove this is the config you'd use:

    Code:
    {% raw %}
    "Remove - Normal - Use Building Owners (You will need Building Owners plugin)": false,
  "Remove - Normal - Use Entity Owners": false,
  "Remove - Normal - Use Friends (Friends)": false,
  "Remove - Normal - Use Friends (RustIO)": false,
  "Remove - Normal - Use Tool Cupboards (strongly unrecommended)": true,
    {% endraw %}
    Why does the Dev say "strongly unrecommended' next to the option you want? Because it's the worst option available, and here's why. The Remove Tool is for building, not raiding. It's not free C4. People in Rust build secure bases with honeycombed layouts to minimize loss during a raid. You might get into one loot room, but you won't get into all of them, not without a lot of C4. With the config above, however once you get to the ground floor TC, the whole base is compromised. Doesn't matter how many walls they've built, or how well they've built. Not only can the attacker destroy the base, but they can actually steal every bit of it.
    So, if you chose to use this option, be prepared for the players on your server to cry and complain for about 5 minutes before they move onto another server.
    If there is something I'm missing, some great reason to use that config, please let me know. I can't find one, but people keep posting the same question.
