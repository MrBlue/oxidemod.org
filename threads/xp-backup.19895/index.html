    PaiN submitted a new resource:

    XP Backup - Backup player's experience.

    This plugin adds the ability for users with permission to save, remove & give player's saved experience.

    Note: Please keep in mind that this plugin only saves experience information for players that are online. If you want to keep everyone's xp info then keep your XP files from your server's identity(usually its my_server_identity) but it may be something else.

    Commands:
    - /xpb <save/remove/give> <player/all> <GiveOnConnect:true/false>\" (ex. /xpb save PaiN true)...

  11. Alphawar

    Hi @PaiN .
    Just wanted to give you a heads up I recently added a big feature to my PvX mod, it now tracks players XP, The reason is to store and restore lost XP when changing levels due to PvP/PvE caps set by the server, But this will cause odd things to happen when mods add xp, not really an issue if just adding, but more when doing something like: agent.reset then agent.add(leveltoXP) as all that xp will get added to the xp from before.
    I have an easy fix via a few simple hooks that mods can call to tell my mod when to ignore a players xp change.
    If you would like to add this support into your mods that affect XP please check out this page, I havent updated my main page yet to reflect this info.
