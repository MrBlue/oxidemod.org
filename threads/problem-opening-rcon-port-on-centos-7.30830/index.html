    hello,
    as of right now im trying to setup rust server, I first did this with firewall disabled and I was able to connect to the server through the Rusty rcon panel. now I decided to enable the firewall and open up the ports for security reasons. I masked Firewalld and unmasked iptables, I tried opening the ports using
    Code:
    {% raw %}
    iptables -A INPUT -p tcp -s {my_ip} -m tcp --dport 28015 -j ACCEPT
iptables -A INPUT -p tcp -s {my_ip} -m tcp --dport 28016 -j ACCEPT
    {% endraw %}
    but whenever I do
    Code:
    {% raw %}
    nmap {my_ip} -p 1-28016
    {% endraw %}
    it returns the following
    Code:
    {% raw %}

    I've tried to run multiple commands but none of those helped me in opening the ports.
    does anyone have an idea or encountered the same issue? thanks in advance

  2. kenna

    # Rust server uses UDP, Rcon uses TCP
    firewall-cmd --zone=public --add-port=28015/udp --permanent
    firewall-cmd --zone=public --add-port=28016/tcp --permanent

    firewall-cmd --reload

    # iptables

    iptables -A INPUT -p udp --dport 28016 -m udp -j ACCEPT
    iptables -A INPUT -p tcp --dport 28016 -m tcp -j ACCEPT

    If you are not familiar with none of them, i find the easy to use is firewall-cmd on centos.

    if it still does not work (iptables) you need to send the output (iptables -L).

  3. nimdasys

    Add the -sU parameter to scan UDP ports.

    Code:
    {% raw %}
    nmap <serverIP> -sU -p 28015
    {% endraw %}
    You probably don't want to include "-s {my_ip}" unless its your intention to only allow that single IP through the firewall.

    This should work:

    Code:
    {% raw %}
     iptables -A INPUT -p udp --dport 28015 -j ACCEPT
 iptables -A INPUT -p tcp --dport 28016 -j ACCEPT
    {% endraw %}
