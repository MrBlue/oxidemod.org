  2. Deicide666ra

    Definitely doable, in fact I'm working on a total tree farming revamp plugin for my server .

    There are a lot of tree models (more than you'd assume just looking at the world, if I remember right there are close to 40 models) and they all vary in rarity, from extremely rare to very common. Some are exceptional finds, there will only be a dozen or so on a server at any given time.

    My plugin doesn't create new trees or change anything to the spawns, it simply changes the gathering rates depending on tree rarity, size and model scale. I'll also introduce a hardness factor to tree models, making oak give more wood at the cost of more condition loss on tools for instance.

    I'm almost done with this in fact, but I do not plan on releasing it, it will be like most of my plugins a private plugin for my server.

    In your case, what I would do is simply use some of the tree models that are not used by the game. From what I can remember there are a couple of models that simply do not spawn at all, you could simply use these as your "super" trees and nerf the gathering on everything else. You could then control how many of those trees exist in the world.

    As far as placement goes I'd use the lazy way and simply replace existing objects like I did in Cornucopia. I would avoid using the trees themselves because just getting the list of trees is time consuming. I'd look once on server start and then keep a count in the plugin itself, increasing each time you spawn a new one and decreasing each time one is downed.

    If you're going for a Legacy feel, you'll probably want to have specific spawn points or perhaps zones, I'd look at the Spawns Database plugin or Zone Manager for this.
