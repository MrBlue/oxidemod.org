  6. CaptainNebulous

    yes sir.

    SecondsPerSpawn - How many seconds it takes for item to be respawned

    SpawnedLimit - Maximum number of spawns on the map (there is only 90 maximum possible I believe)

    InitialWaitTime - How long after the count goes below the minimumspawncount before it spawns to fill up to minimumspawncount

    MinimumSpawnCount - Plugin will spawn as many as needed to reach this number after ___ seconds (initialwaittime)

    Probability - Not 100% sure what this value does, doesn't seem very important to use. I'd keep it at the default value. It definitely effects how common loot crates are, but I noticed you get a lot more customization results when keeping it at 1 and adjusting the previous values. Remember there are only 90 possible spawns for this item so not much room to work with with these "broader" functions. I think it's some sort of broad setting that effects everything, but if that's the case.... just like with stacksizemultiplier in the spawnconfig plugin.... in this case the math doesn't add up quite right to use it in a balanced manner.
