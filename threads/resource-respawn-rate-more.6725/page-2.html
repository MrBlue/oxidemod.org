    Yes cdump only works with the plugin but take time to read what I said.. Run the plugin with the default config, this will not spawn any extras or cut down any items either so after 30 mins of running like this (you'll need to restart the server if you had another config before), the population of items will be "normal" and /cdump will show you the "normal" numbers for your map.

    Once you got that, change the MIN values for items for which you don't have enough.. OR the MAX value if there are any items you feel you have too much of.

    The MAX value is very situational... You could use it to remove chickens if they bother you, or limit the number of a resource if you feel there is too much of it, etc.

    Never use BOTH MIN and MAX there is no point whatsoever.

  15. -[KFS]- X.W.A.R.R.I.O.R

    UserUnknown said: ↑
    Put in console:
    • spawn.min_rate # (default .1)
    • spawn.max_rate # (default 1)
    • spawn.min_density # (default .1)
    • spawn.max_density # (default 1)

    This changes spawn rates and density for ALL gather-able entities (eg; trees, ores, animals) and no, you CAN NOT choose which entities to change the rate/density individually, that would/could be a plugin/extension required to be written to be able to choose individual entities (good luck at that).

    Advice: Do not set too high for too long as it can spawn entities like crazy and end up bringing your server almost to a halt.
