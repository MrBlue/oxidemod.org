  2. sqroot

    When starting the server, Rust first checks if a save file with the specified seed and the specified size can be loaded.
    If it can, it uses that map.
    If it can't, it generates a new one and saves the new map in the save directory.
    This means that just changing startup parameters is enough to use a different map.
    If you would like to keep the current map but wipe it, this means that you have to delete the existing save file for the respective size/seed.

    It kind of sounds like your fresh install is broken, using a different map and getting a simple Rust server up and running is pretty straight forward.

    It actually sounds like you are trying to specify the map as +server.map Procedural Map, without quotation marks, which will cause the error shown. If using that map, you can either no use the option as it is default or use +server.map "Procedural Map" with quotes.
