    In the plugins current form of Forcing a players position to allow a simulated hack noclip, its about as good as I can get it. (without spectate)

    But, I am playing with giving players temp admin status to actually use the admin noclip.

    Initial Method 1 - When players typed /noclip..it gave players a IsAdmin flag. And activated the admin noclip for them. Seemed to work perfect. Until timing of admin flag removal and disabling of noclip gets wacky with lag or glitch or another plugin. Then players would uncommonly get insta banned from that server for Anithack : cheat. no fun. gotta unban folks all the time.

    Method 2 - giving players IsAdmin flag after login, then removing it when disconnecting. Actually so far. works great. So this one looks promising. Still testing though. Players can bind the noclip function like admins, but they have no access to any other commands (as far as me and my players can tell). As players are not logging in as admin, they are just getting flagged as admin after they login. Only drawback for this method, and method 1, so far is if a plugin used the IsAdmin flag to authenticate within the plugin, then players would get that access too


    So maybe something will prove to be reliable and safe to use for a better FauxClip or maybe facepunch will move the noclip command to check server side
