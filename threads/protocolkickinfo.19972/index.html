  1. FuJiCuRa

    Fujikura submitted a new resource:

    Protocol Kick Information - Customize disconnect info for being kicked by wrong protocol version

    This plugin does provide a simple solution to give users a clear information, why they get kicked, mostly direct on patch-days.

    The plugin supports both reason types:
    • client protocol higher then server
    • client protocol lower then server
    The messages can be changed by the provided language file.

    As a result, it will produce a direct feedback like this:

    View attachment 25792
    View attachment 25793

    The language file content:

    Code:

  "msgServerWrong":...
