    Sorry about that, As the mod was uploaded about 3 weeks back and started creating over 2 months back I hadn't finished getting the notes finished.
    In the simplest statement all PvP action is not allowed, this mod was designed to help large role playing servers not have to deal with complaints of RDM.
    For players who want to have some form of pvp action can then use the /pvp or /raid command. How the /pvp command works is as follows.
    PvP
    1. Player enters in command
    2. Mod checks every player within PvPZone1, using RustIO it checks if any player in that zone are not friends.
    3. If there is a player in that zone who are not friends (tests boths sides) then it will scan for players in PvPZone2.
    4. Any player located in PvPZone2 will be register to that specific pvp zone.
    5. After a set "cooldown" time that pvp zone will then become active.
    6. When a player then attacks another player the mod checks a few things for the PvP zone.
      1: are both players in the zone (Governed by PvPZone3).
      2: are both players registered to that zone.
    7. If no zone passes these checks then no damage is done, if at least one zone passes these checks damage is done.
    Raid
    1. Player enters in command
    2. Mod checks every building block within RaidZone1, using entity.owner and RustIO to check for friends.
    3. If there is any building within that zone that does not share a friend link with that player it will continue
    4. Mod will create a Raid zone at the point of the player
    5. Any player who does damage to another player or entity will then get checked for the following:
      1:Are both players (or player and entity) in the Raid Zone (Governed by RaidZone2)
    6. If this check is passed damage will be done, if not then damage is nuliffied
    Now both zones will only last as long as the pvpTimeLimit and RaidTimeLimit, I have added a function to the mod that resets the timer though so in a pvp zone if someone died who was registered to that zone then the zone timer will be reset, for the RaidTimeLimit there is an extra feature, it will reset when a player dies but will also reset when a structure is destroyed.

    As stated in the mods info this mod was designed for a large Role play server who was dealing with RDMers and this was designed to counter those bandits to allow normal players a change to escape.
