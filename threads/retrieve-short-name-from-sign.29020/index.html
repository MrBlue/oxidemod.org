    You can't get the item shortname from the sign directly, you can however get the name of the prefab (full or short) allowing you to identify the type of the sign. If you'd wish to get to the item shortname from this you will have to map the itemshortnames and prefabs when the server is first initialized so you can lookup the item shortname by the prefab name from there.

    The FileStorage class uses the networkable id for identification as well (sign.net.ID)
