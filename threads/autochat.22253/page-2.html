    - First, the "PrivateMessage" that is in the optional dependencies list is the one supported and if you put another with a different name the plugin doesn't see it. If you want to put a different private message plugin, you can add it to the "CustomChat" configuration (see example in the overview). Remember to reload the plugin.
    - If you still have problems, try deleting the config and data, reload the plugin and check if you see the message in console (Supported plugins: ...). If the supported plugins come out it means it works and you just have to edit the config and enable autochat ("/ac active" if the player wants to use it but you can edit "PlayerActive" on true to activate it on automatically when the player enters for the first time).
    When it is active, it show the UI near game chat, so try sending a pm and see if "g" becomes "pm".
    - If you use a plugin different than "BetterChat" for chat customization, it may be a cause of the problem. Send me the link of the plugin you are using and I will try to solve the problem, but if it's a private plugin, obviously, I can't do anything without seeing the code.

    Permission to add:
    grant group defaul autochat.use

    If none of these things work, send me the "Supported plugins" message and the plugins you are using.
