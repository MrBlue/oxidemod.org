    The only cause would be mixing Rust versions. This would either be from manually downloading an older build (or cached by your browser) else your host providing an older build. There is nothing we can do to resolve the issue, the builds we are providing are updated for the latest Rust release (not staging, not development) build as of 3/29 per the changelog I posted the day of that update.

    Again, this is caused by mixing Rust versions, which happens when either not fully updating the Rust server before installing an newer Oxide build that contains patched Rust DLLs meant only for the version it was patched for, else updating the Rust server and using an older Oxide build that contains patched Rust DLLs meant only for the version it was patched for.

    So, if you are using a host that provides the install method, you'd need to verify with them that they are using an updated link and that it is downloading it properly (they should verify the hash against what our builds on GitHub.com have). If you are downloading manually, delete all existing copies of Oxide-Rust.zip Oxide.Rust.zip you may have downloaded and then download again from a fresh browser instance (or private) from the provided links here are on our site or at OxideMod/Oxide.Rust. Also make sure that you are using the "validate" option when updating the Rust version; and when installing Oxide, make sure to overwrite ALL files when prompted.
