    Hello,

    Me and my friends have recently bumped into one very peculiar issue with our Rust Server.
    We recently aquired a new dedicated server for our proxmox cluster and decided to load a VM with Rust onto that new node.
    Each node on the cluster has 2 networks, public and internal, the Rust VM is running on the internal network and has port forwarding setup between the public network and internal network.

    Now to the problem.
    For me and 1 more friend we can connect without any issues at all and play, for 3 other friends it does not work at all.
    They can ping the server, but the server is not visible through the steam server browser (adding manually in favorites), they get the message "Server is not responding" in Steam.

    We found a workaround that resolves this for now, if they connect to the RCON Console (with Rusty) they can then connect to the server without any issues.

    Googling this issue does not help becasue 99.9% of the threads (on any public resource) is by people that has not portforwarded correctly.

    This cluster is located in country A, I am in country B and friend in country C.
    We are all using different ISP's and completely seperate networks.

    My friends have tried the following without success (this was done before testing rusty):
    • client.connect X.X.X.X:28015
    • steam://connect/X.X.X.X:28015
    • ipconfig /renew
    • ipconfig /flushdns
    • Reboot of the computer (duhh..)
    • Reboot of router
    • Reinstallation of Rust
    • Reinstallation of Steam
    • Firewall disabled
    • AntiVirus disabled
    • Running Steam as admin
    • Running Rust as admin
    • Firewall on server has been disabled
    • Rust VM was moved from internal network to public.
    • Rust VM was recreated from scratch
    • DNS records were recreated
    I might have forgotten something here and I am definately overlooking something.
    Any questions, ides, tips or suggestions on what could be at fault is greately appretiated.

    Thanks in advance!
