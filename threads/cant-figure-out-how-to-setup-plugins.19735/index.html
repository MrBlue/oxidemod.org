  3. sqroot

    Permissions exist so that you can configure to whom certain plugin features apply.
    Other games that allow you to configure this likely have a similar system as well.
    Plugins usually default to something sensible (Example: Built for Rust | Oxide, the built.use permission is given to admins by default).
    If they don't and the permissions of the plugin are hard to configure, it's usually a shortcoming of the plugin, not of Oxide itself.
    I agree that most plugins don't take defaults seriously enough and make you configure a lot to make stuff work sensibly, but that's an issue that can occur in any modding community, not just Oxide, and isn't related to Oxide.

    You could also provide some more information so that we can help you.
    • What commands did you try?
    • What plugins are difficult to configure for you?
    • Which permissions are you trying to configure?
    • Since you are saying that nothing is working, what else, other than permission commands not being recognized, doesn't work?
