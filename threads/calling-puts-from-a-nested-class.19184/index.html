Discussion in 'Rust Development' started by Yatta, Jul 4, 2016.

  1. Yatta

    Quick dumb question as im not familiar with c# scopes.

    If i create something like that :

    Code:
    {% raw %}
    namespace Oxide.Plugins {
    [Info("Equity", "Yatta", "0.0.1")]    class Equity : RustPlugin {
        static string test = "whatever";
      
        public class SomeClass : Monobehavior {
            test = "I cant even";
            Puts(test);
        }
    }
}
    {% endraw %}
    I can change the variable "test" from SomeClass because its static, but I cant access "Puts" anymore. How can I mae it so that I can use Puts and other common methods from my custom class ?

  2. Wulf Community Admin

    You'd need to use another method other than Puts I believe, likely calling it directly. Oxide's API only really supports the main class. You could also call Unity's logging methods directly if needed.

  3. Mughisi

    You can store an instance of your plugin in a variable and use that to call puts.

    So basically define a "public instance <InsertPluginClassName> Instance;" in the plugin's scope and initialize it with the plugin as value. Then in your class you call "<InsertPluginClassName>.Puts("message");"
